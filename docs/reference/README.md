# Reference

Authoritative specifications. These define the stable contracts third parties build
against — domain-pack authors, adapter implementers, and integrators.

| Document | Purpose |
|----------|---------|
| [Domain-Pack Specification](./domain-pack-spec.md) | The declarative manifest format that onboards a brand-new market — schemas, ontology, quality rules, models, policies. |
| [Adapter & Service Interfaces](./adapter-interfaces.md) | Every adapter and service protocol, the lite reference implementations, and how to add a new backend without touching the core. |
| [Access Control & RBAC](./access-control-rbac.md) | Identity, roles, ABAC policies, the super-admin apex role, and how authorization is enforced end to end. |

> These map directly onto the frozen contracts in
> [`engine/aegoria_core/contracts/`](../../engine/aegoria_core/contracts/). When a spec and
> the code disagree, the code is the source of truth.
